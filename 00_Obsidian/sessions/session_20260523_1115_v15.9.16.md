## 📌 변경 사항 요약
- **보안 취약점 원천 해결**: 소스 코드 내부 및 `workspace_state.json`에 하드코딩 형태로 포함되어 유출의 빌미가 되었던 실제 제미나이 API Key 텍스트를 제거하고 공란으로 정제했습니다.
- **보안 설정 로컬 격리 분리 (`local_secrets.json`)**:
  - 민감한 보안 인증 정보 5종(`gemini_api_key`, `tavily_api_key`, `youtube_api_key`, `github_token`, `github_pat`)을 기존 `workspace_state.json`에서 완전히 분리시켰습니다.
  - 중요 키들은 오직 로컬 디스크에만 저장되고 Git 감지 대상에서 원천 배제되는 `local_secrets.json`에 따로 격리 보관하도록 변경했습니다.
  - 앱 기동 시 두 설정 파일을 자동으로 안전 병합하여 읽어오도록 `load_workspace_state()`를 개조하여 사용자의 UX 편의성과 인증 상태는 그대로 유지되도록 설계했습니다.
- **.gitignore 이중 보호**: `.gitignore` 파일에 `local_secrets.json`을 명시적으로 차단 등록하여 실수가 있더라도 깃허브에 평문 키 정보가 다시는 업로드되지 못하도록 방화벽을 적용했습니다.

## 🛠️ 수정 및 추가된 함수 목록
- `save_workspace_state()` : 민감 정보 필터링 및 `local_secrets.json` 별도 저장 연동
- `load_workspace_state()` : `workspace_state.json`과 `local_secrets.json` 자동 병합 로딩 연동
- `init_session_state()` : 기본 설정 상태 내 하드코딩 키 제거
- 사이드바 설정 텍스트 입력창 컴포넌트 : default value 하드코딩 키 제거
