# Sage Mirror Studio Session — v16.1.6 (2026-05-26 08:38)

## 📌 변경 사항 요약
- **Agent Toolkit 모듈 분리**:
  - `sage_popups.py` 및 `app_v16_1_5.py` 내부에 혼재되어 있던 에이전트 도구 실행 전후의 결과 문자열 포맷팅, 도구 이름 정규화, 명령 파싱, 지원 도구 목록 등의 비-UI 헬퍼 로직을 신규 순수 Python 모듈 `agent_toolkit.py`로 격리하여 분리했습니다.
  - `agent_toolkit.py`는 순수 Python 모듈로서 Streamlit UI 및 세션 상태 접근을 완벽히 차단하여 플랫폼 독립적인 코어 모듈로 구성했습니다.
  - 신규 버전 `app_v16_1_6.py`를 제작하여 버전 명을 `v16.1.6`으로 갱신하고, 기존 에이전트 루프 및 동작의 정밀성을 완전히 보존했습니다.
  - `sage_popups.py`에서 `agent_toolkit` 모듈을 연동하여 `_detect_tools`가 도구 정규화를 사용하고, `_execute_tool`이 개별 결과 포맷터를 호출하여 처리하도록 리팩토링했습니다.
  - 배포 스크립트 3종(`RUN_APP.bat`, `RUN_DEBUG.bat`, `RUN_APP.vbs`)의 Streamlit 기동 파일 실행 대상을 `app_v16_1_6.py` 및 `8505` 포트로 갱신했습니다.

## 🛠️ 수정 및 추가된 함수 목록
- **`agent_toolkit.py`** [NEW]
  - `get_supported_agent_tools() -> dict`: 지원하는 도구 목록 및 정규식 패턴 사전 반환
  - `normalize_tool_name(name: str) -> str`: 도구 명 정규화 (예: `NEED_RESEARCH` -> `SEARCH_WEB` 등 클렌징)
  - `parse_tool_command(text: str) -> dict | None`: 텍스트 내 도구 태그 파싱 헬퍼
  - `format_tool_result(tool_name, success, data, error_msg) -> str`: 도구 공통 실행 결과 래퍼 포맷터
  - `format_check_source_result(param, results, citation_formatter) -> str`: 출처 검증 결과 포맷팅
  - `format_search_web_result(query, results, results_formatter) -> str`: 웹 검색 결과 포맷팅
  - `format_search_youtube_result(query, results) -> str`: 유튜브 검색 결과 포맷팅
  - `format_save_obsidian_result(param, success, error_msg) -> str`: 옵시디언 자동 저장 결과 포맷팅
  - `format_save_reference_result(param, success, error_msg) -> str`: Reference 저장 결과 포맷팅
  - `format_build_packet_result(param, success, error_msg) -> str`: 패킷 빌드 결과 포맷팅
- **`app_v16_1_6.py`** [NEW]
  - 상단 헤더 버전을 `v16.1.6`으로 갱신하고 동작 안정성 유지
- **`sage_popups.py`** [MODIFY]
  - `_detect_tools` 함수에서 `agent_toolkit`의 `get_supported_agent_tools()` 및 `normalize_tool_name()`을 활용하여 도구 검출 및 이름 정규화 적용
  - `_execute_tool` 함수에서 문자열 포맷팅 및 에러 핸들링 출력 부분을 `agent_toolkit`의 포맷터 함수들(`format_search_web_result`, `format_check_source_result`, `format_save_obsidian_result`, `format_tool_result`)로 이관
