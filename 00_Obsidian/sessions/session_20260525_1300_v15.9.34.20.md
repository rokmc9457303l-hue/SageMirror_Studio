## 1. 변경 사항 요약
- **Gemma 어시스턴트용 RAG 메모리 블록 빌더 구현**: References 로더 결과 리스트를 가공하여 젬마 프롬프트 컨텍스트에 삽입 가능한 통합 RAG 기억 장치 블록을 조립하는 `build_gemma_memory_prompt_preview()` 함수 구현. 프롬프트 블록의 최종 크기가 `max_chars=30000`자를 초과할 수 없도록 설계했으며, 초과 시 본문을 안전하게 절삭하고 `[메모리 일부 생략됨 — max_chars 제한]` 안내 문구를 자동 결합하는 에러 방지 패턴 적용.
- **수동 프롬프트 미리보기 UI 연동**: 최하단 테스트 영역에 "🧠 Gemma Memory Prompt 미리보기" 컴포넌트를 추가하고, 사용자가 직접 버튼을 클릭했을 때에만 실시간으로 메모리를 가공해 생성된 프롬프트 문자열 전문을 대용량 `st.text_area` 컴포넌트(높이 500px)에 출력하도록 연동.
- **임포트 및 배포 스크립트 갱신**: `sage_popups.py` 내 `save_workspace_state` 임포트 참조를 `app_v15_9_34_20`로 교체하고, `RUN_APP.bat` 및 `RUN_DEBUG.bat` 배치 파일들의 구동 대상을 `app_v15_9_34_20.py`로 변경.
